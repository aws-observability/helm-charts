# Custom Resource Definitions

## Purpose
CRDs that define the operator's API surface. These are generated artifacts from the operator source code — do not hand-edit.

## CRDs
- `AmazonCloudWatchAgent` — Core agent workload. Supports Deployment, DaemonSet, StatefulSet modes. Manages the CloudWatch Agent lifecycle including config, scaling, updates, and TLS.
- `Instrumentation` — OpenTelemetry auto-instrumentation spec. Configures SDK injection for Java, Python, .NET, Node.js, Apache HTTPD. Controls sampling, exporters, resource attributes, and propagators.
- `DcgmExporter` — NVIDIA GPU metrics exporter. Operator reconciles this into a DaemonSet.
- `NeuronMonitor` — AWS Trainium/Inferentia accelerator monitor. Operator reconciles this into a DaemonSet.

## Anti-Patterns
- Never edit these files. They're generated by `controller-gen` (noted in annotations). Changes will be overwritten.
- Don't add new CRDs here — they come from the operator repo.
- If you need to understand a CRD's full schema, read the file, but make changes in the operator source.

## Notes
- `AmazonCloudWatchAgent` uses short names `otelcol`/`otelcols` (inherited from OpenTelemetry Collector operator lineage).
- `Instrumentation` uses short names `otelinst`/`otelinsts`.
- All CRDs are `Namespaced` scope, API group `cloudwatch.aws.amazon.com`, version `v1alpha1`.
